## 一、比赛概况

人类对话天然是**全双工**的：我们能在倾听的同时组织语言，在说话时思考，预判对方的停顿并自然接话，用简短的反馈表明「我在听」，也能在必要时礼貌地打断。这是人类社交互动的核心基石。

随着 GPT-4o、Gemini Live 以及 Moshi、Freeze-Omni 等开源模型的出现，语音交互已进入**原生音频（Native Audio）**时代。全双工模型打破了传统 **ASR → LLM → TTS** 的串行链路，实现端到端建模，将延迟压缩到人类感知阈值以内。然而，如何准确判断「何时开口」「何时收声」，仍是决定系统体验的核心瓶颈。

**第11届信也科技杯全球 AI 算法大赛**聚焦**多方言中文对话的轮次交互建模（Turn-Taking）**：基于双方对话的音频历史，预测在指定时间窗口内是否会出现相关语音事件，使 AI 能以低延迟判断用户意图，避免过于激进或造成尴尬沉默。大赛诚邀全球技术精英，探索多模态信号，帮助 AI 获得「社交直觉」，实现自然流畅的人机交互。

## 二、赛题介绍

### 2.1 概念介绍

数据由**用户 A（说话方，上图/红色波形）**与**用户 B（倾听方，下图/绿色波形）**之间的对话构成。音频按连续的 **80ms** 片段切分为 **chunk**，每个 chunk 标注以下五类事件之一：

| 标签 | 含义 | 说明 | 示例 |
|------|------|------|------|
| **C** | Continuation（延续） | 当前说话方继续发言 | 一方在陈述观点 |
| **T** | Turn Change（话权转移） | 说话权转移给对方 | 「你觉得呢？」→ 对方开始说话 |
| **BC** | Backchannel（附和反馈） | 轻量反馈，不夺取话权 | 对方说话时插入「嗯」「对」 |
| **I** | Interruption（打断） | 双方同时说话，争夺话权 | 一方打断另一方 |
| **NA** | Silence（静音） | 双方均不说话 | 话题之间的短暂停顿 |

**典型对话片段示意：**

- **NA**：双声道均为静音
- **T → C**：用户 A 停止，用户 B 开始（话权转移后继续）
- **BC**：用户 A 说话时，用户 B 发出短促附和音
- **I**：双方波形重叠，发生打断

---

### 2.2 预测任务

**任务定义：** 给定过去 **30 秒**的双声道音频作为上下文，预测未来 **25 个 chunk（共 2 秒）** 内的语音事件。

```
┌─────────────────────────────────────┬──────────────────────────┐
│         上下文窗口（30 秒）           │   预测窗口（2 秒）        │
│    已完成的双声道音频                │   25 个 chunk            │
│                                     │  chunk1 … chunk25        │
└─────────────────────────────────────┴──────────────────────────┘
                              ↓
                    模型输出每个 chunk 的事件类别
                    （0|1|2|3 对应 C|T|BC|I，另有 NA=4）
```

- **输入**：30s 双声道对话音频（上下文）
- **输出**：预测窗口内 25 个 chunk 各自对应的语音事件类别

---

### 2.3 数据集构成

#### 训练集

| 项目 | 说明 |
|------|------|
| **来源** | 真实双声道电话对话数据 |
| **内容** | 完整双声道音频（`.wav`）、chunk 级标签序列、对应文本转写（`.json`） |
| **构造方式** | 官方 baseline 将数据切分为固定的 **30s 上下文 + 2s 预测窗口**；标签基于 VAD 与对话规则生成五类（C、T、BC、I、NA）。参赛者可从原始数据自行构造训练样本 |

**文件结构：**

| 路径 | 说明 |
|------|------|
| `audio/<conv_id>.wav` | 整通双声道对话音频 |
| `text/<conv_id>.json` | 整通对话的文本转写 |
| `labels/<conv_id>.npy` | 序列标签：`0`(C)、`1`(T)、`2`(BC)、`3`(I)、`4`(NA) |

#### 测试集 1 — 公榜测试集

- 从数据中**随机、无重叠**抽取 **30s 上下文 + 2s 预测窗口** 样本
- 打乱、匿名化后划分为两个子集
- 用于**公榜**排名

#### 测试集 2 — 私榜测试集

- 为上述划分的**另一子集**
- 可能包含额外私域业务数据
- **关键差异**：上下文长度**动态变化**，范围为 **(0, 30] 秒**，而非固定 30s；官方维护映射与标签分布以保证评测公平

## 三、与本仓库 Baseline 的对应关系

本仓库实现「**过去 30s 音频 + ASR 文本 + 历史标签序列 → 预测未来窗口内话权相关事件**」的多模态模型，默认预测未来 **2s（25×80ms chunk）** 内五类标签是否出现。详细环境与训练说明见项目根目录 [README.md](../README.md)。
